## Venv decision + rationale

**Decision: fresh `.riptide-fx-venv` (matches plan E3 — no deviation).**

The plan's E3 wanted a fresh `.riptide-fx-venv`; the spike venv (`.riptide-spike-venv`) already had
pytest 9.1.0 + numpy 2.4.6, so reuse was the fallback. The fresh venv was attempted first **offline**
and it worked completely from the uv cache — `uv venv .riptide-fx-venv` then
`uv pip install --offline pytest numpy` resolved and installed 6 packages
(`pytest 9.1.0`, `numpy 2.4.6`, `pluggy 1.6.0`, `iniconfig 2.3.0`, `packaging 26.2`, `pygments 2.20.0`)
in ~45 ms with **no network access**. Because the canonical fresh-venv path succeeded offline, there
was no reason to fall back to reusing the spike venv — the plan's preferred E3 is satisfied exactly.

- Both venvs run CPython **3.14.4** (≥3.12 as required) and import `pytest`, `numpy`, `sqlite3`.
- The fresh venv keeps Phase 3 deps isolated from the Phase 1 spike venv; system Python untouched.
- `.riptide-spike-venv` remains available as a verified fallback if `.riptide-fx-venv` is deleted.

## Corpus: location, collected count, pytest result

- **Location:** `benchmarks/fixtures/fx_corpus/`
- **Generator (deterministic):** `benchmarks/fixtures/fx_corpus/generate_fx_corpus.py`
- **Collected count N = 511** (`pytest --collect-only -q` → `511 tests collected`)
- **Run result:** `pytest -q` → **`511 passed`**
- **Determinism:** regenerating the tree produces a byte-for-byte identical result (verified by
  hashing all `*.py`/`*.ini` before and after regeneration — hashes match). No timestamps, no
  randomness. Runtime instrumentation artifacts (`events.log`, `counts.json`) are written to a
  per-run temp dir chosen by the `probe_dir` session fixture, **never** into the committed tree.

### Fixture features exercised (each as real pytest that passes)

| Required feature | Where in `fx_corpus` |
|---|---|
| session / package / module / class / function scopes | `conftest.py` (session_db, session_autouse), `tests/conftest.py` (pkg_resource, pkg_autouse), `tests/test_scopes.py` (module_fix, class_fix, func_fix) |
| **autouse** fixture in a conftest | `session_autouse` (session, root conftest) + `pkg_autouse` (package, `tests/conftest.py`) |
| **override-by-location** (module shadows same-named session/conftest fixture) | `shared_value` defined in root `conftest.py` (session) and overridden in `tests/pkg_override/test_override.py` (module); `test_override_wins` asserts module value wins |
| **yield teardown / finalizers** with observable ordering | every fixture is `yield`-style and calls `setup()`/`teardown()` which append ordered lines to a shared `events.log`; teardown order is the strict reverse of setup order |
| **parametrized fixtures** (`params=[...]`) | `tests/test_param.py` — `parametrized` fixture `params=['a','b','c']`, 3 instances, 3 invocations |
| **non-fork-safe resource** (`sqlite3.connect(":memory:")`) with detectable intent | `tests/test_sqlite_resource.py` — fixture name prefixed `reinit_after_fork__` **and** tests marked `@pytest.mark.reinit_after_fork` (both detectable by the Rust side) |
| **scope-count probe** (module/session body runs once; function per test) | `fx_probe.py` `bump_count()` → `counts.json`; verified: `session_db`=1, `module_fix`=1, `class_fix`=2 (two classes), `func_fix`=3 (per test), `parametrized[a/b/c]`=1 each, `reinit_after_fork__db_conn`=2 |
| **~500 trivial tests sharing one module fixture** ("1× not 500×") | `tests/test_big_module.py` — 500 tests share `big_module_fix` (module scope); probe confirms `counts.json["big_module_fix"] == 1` |
| Real C-extension warm at snapshot (E5 hazard) | `tests/test_numpy_cext.py` — module fixture builds a numpy array; numpy is warm when the wellspring would snapshot |
