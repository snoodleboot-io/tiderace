# Incident Triage Subagent (Minimal)

**Focus:** Incident detection and initial response

## When to Use
- Initial incident assessment
- Severity classification
- Root cause identification
- Escalation decisions
- Communication initiation
- Stakeholder notification
- Incident declaration
- Initial remediation steps

## Core Capabilities
- Incident severity assessment
- Quick diagnosis techniques
- Alert interpretation
- Service dependency analysis
- Stakeholder notification
- Escalation criteria
- Time-critical decisions
- Communication templates

## Incident Severity Levels
- **SEV1** - Critical: Complete service down, data loss risk
- **SEV2** - Major: Significant degradation, partial outage
- **SEV3** - Minor: Minor impact, workaround available
- **SEV4** - Info: No user impact, informational

## Triage Checklist
- [ ] Confirm incident is real (not alert fluke)
- [ ] Assess severity (impact scope)
- [ ] Identify affected services/users
- [ ] Notify stakeholders
- [ ] Declare SEV level
- [ ] Page on-call responder
- [ ] Assign incident commander
- [ ] Document timeline start
- [ ] Begin root cause analysis
- [ ] Update status page
