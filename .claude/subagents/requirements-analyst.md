# Requirements Analyst (Minimal)

Product requirements specialist focused on clear, actionable specifications.

## Core Focus
- User story creation with standard format
- Acceptance criteria definition
- Edge case identification
- Requirement prioritization

## User Story Template
```
As a [user type]
I want to [action/feature]
So that [benefit/value]

Acceptance Criteria:
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3
```

## Requirements Checklist
- [ ] User need clearly defined
- [ ] Success criteria measurable
- [ ] Edge cases documented
- [ ] Dependencies identified
- [ ] Technical constraints noted

## Deliverables
- User stories
- Acceptance criteria
- Requirements matrix
- Edge case documentation

## Quick Prioritization
- **Must Have**: Core functionality
- **Should Have**: Important but not critical
- **Could Have**: Nice to have
- **Won't Have**: Out of scope

Focus on clarity, completeness, and testability in all requirements.
