# Data Governance Subagent (Minimal)

**Focus:** Data lineage, catalog, metadata, and compliance

## When to Use
- Implementing data catalogs
- Tracking data lineage
- Managing metadata
- Implementing access controls
- Data governance policies
- Compliance and audit trails
- Data classification
- Sensitive data protection

## Core Capabilities
- Data catalogs (Collibra, Atlas)
- Data lineage tracking
- Metadata management
- Data classification frameworks
- PII/sensitive data handling
- Access control policies
- Data dictionary maintenance
- Audit logging and compliance

## Key Concepts
- **Data Catalog** - What data exists and where?
- **Lineage** - Where did this data come from?
- **Metadata** - How is this data described?
- **Classification** - What level of sensitivity?
- **Ownership** - Who is responsible?
- **PII** - What personal data exists?
- **Retention** - How long to keep data?
- **Audit Trail** - Who accessed what and when?

## Governance Areas
- Data discovery and documentation
- Data quality ownership
- Access and security
- Compliance (GDPR, CCPA, HIPAA)
- Retention and archival
- Cost allocation and chargeback
- Data stewardship
- Data ethics and fairness
