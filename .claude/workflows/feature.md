## Steps

### Step 1: Before Writing Code

- Restate goal in your own words
- Read relevant source files
- Identify files to change
- Propose approach with tradeoffs
- Flag assumptions
- Wait for confirmation

### Step 2: After Confirmation

- Follow core conventions exactly
- Match patterns in existing code
- Add inline comments for non-obvious logic
- Add TODO for judgment calls
- Implement one file at a time

### Step 3: After Implementation

- List follow-up work (tech debt, missing tests, related changes)
- List tests that should be written or updated

**Output order:** plan → confirmation → implementation → follow-up list
